| ID | Term | Definition |
|----|------|------------|
| T1 | **$P2P** | Protocol native token; sole token for governance voting |
| T2 | **exchangeToken** | Configurable token staked by merchants as liquidity (per Circle) |
| T3 | **settlementToken** | Final token for order settlement (stablecoins only) |
| T4 | **CAIP** | Circle Admin Insurance Pool; first-line coverage (0.1% volume + slashes) |
| T5 | **CALR** | Circle Admin Locked Rewards; 33% of admin revenue locked 30 days |
| T6 | **PIP** | Protocol Insurance Pool; backstop for systemic failures |
| T7 | **RP** | Reputation Points |
| T8 | **CoT** | Circle of Trust |
| T9 | **Proposal** | Unified on-chain request (dispute/claim/audit); OpenZeppelin Governor-based |
| T10 | **Claim** | Unified on-chain request for insurance claim; OpenZeppelin Governor-based |
| T11 | **Dispute** | Unified on-chain request for dispute resolution; OpenZeppelin Governor-based |
| T12 | **T1-T3** | Escalation levels: CoT Admin (T1) / Jury (T2) / Governance (T3) |
| T13 | **U0-U3** | User RP tiers (0-800+): unlock order limits based on formula |
| T14 | **M0-M4** | Merchant RP tiers (0-50k+): unlock stake capacity and benefits |
| T15 | **A1-A5** | Admin RP tiers (9k-45k+): map to Circle capacity tiers |
| T16 | **J1-J4** | Juror RP tiers (1.5k-15k+): unlock dispute value eligibility |
